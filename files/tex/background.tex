Transmission Control Protocol (TCP) uses a three-handshake message exchange before a connection is established. It requires three sets of message exchanges: (i) SYN message from Client to Server stating its desire to connect , (ii) SYN-ACK reply message from Server to Client, approving the connection request and (iii) ACK message from Client to Server, which finalizes the connection between the two. In our project, we will look deeper into three different attacks: TCP SYN Flooding Attack, TCP RST Attack, and TCP Session Hijacking.These three attacks take advantage of the initial exchange of handshake messages. We will go over each attack, discussing what they are, demonstrating the attack, and what mitigation techniques there are for these attacks.
