Transmission Control Protocol (TCP) has different important parts that allow it to operate. For the project, the parts that are important to discuss are the TCP Handshake Process, TCP 4-Tuple and Sequence Number. The TCP Handshake Process is a three-handshake message exchange that occurs before a connection is established. It requires three sets of message exchanges: (i) SYN message from Client to Server stating its desire to connect , (ii) SYN-ACK reply message from Server to Client, approving the connection request and (iii) ACK message from Client to Server, which finalizes the connection between the two. The TCP 4-Tuple is the four unique identifiers of TCP which are Client IP, Client Port, Server IP and Server Port. And finally, the TCP sequence number is the sequence number that is assigned to each packet to keep them in-order during communication. 

In our project, we will look deeper into three different attacks: TCP SYN Flooding Attack, TCP RST Attack, and TCP Session Hijacking.These three attacks take advantage of these different parts of TCP. In each section, it will go over each attack, discussing what they are, demonstrating the attack, and what mitigation techniques there are for these attacks.
